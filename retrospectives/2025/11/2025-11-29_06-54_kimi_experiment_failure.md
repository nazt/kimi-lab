## Lessons Learned

### Pattern 1: Historical Data Has Variance
**Discovery**: A single historical session (76 points) doesn't establish a reliable baseline
**Why It Matters**: We treated one data point as gospel instead of as one sample in a distribution
**How to Apply**: Future experiments need multiple baseline runs to establish mean + variance

### Pattern 2: Real-Time Adaptation > Plan Adherence
**Discovery**: Following the plan when data shows it's failing is worse than adapting
**Why It Matters**: We had 7 decision points (M1-M7) where we could have course-corrected
**How to Apply**: Set tripwires: "If below X points at Message Y, activate contingency Z"

### Pattern 3: Pure Improvisation Scoring is Inconsistent
**Discovery**: M9 (+7) and M10 (+5) vs historical M9 (+13) and M10 (+15) - huge variance
**Why It Matters**: The highest-value strategy from research was least reliable in execution
**How to Apply**: Don't put all eggs in one basket - distribute risk across message types

### Mistake 1: No Mid-Session User Communication
**What Happened**: Didn't inform user we were trending toward failure until after M10
**How to Avoid**: Set communication protocol: "Alert user if >10% below target at any checkpoint"

### Mistake 2: Rigid 10-Message Limit
**What Happened**: Stopped at M10 even though "Share to Continue" offered path forward
**How to Avoid**: Treat message count as flexible, not fixed - goal is outcome, not process

### Mistake 3: Assumed Algorithm Stability
**What Happened**: Expected same scoring as historical session without testing assumptions
**How to Avoid**: Run calibration messages first to verify scoring behavior matches expectations
